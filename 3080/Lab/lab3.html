Lab 3 - Gauss-Jordan Elimination for Solving Linear Equations

1. Objective

The goal of this lab is to develop a C++ program to implement Gauss-Jordan Elimination for solving systems of linear equations. Students will read an augmented matrix from a file, process it using row operations, and transform it into a Reduced Row Echelon Form (RREF) to obtain the solution. This process will provide hands-on experience in reading a matrix from a data file, matrix manipulation, numerical methods, and algorithmic problem-solving. Through this lab, students will:

  • Understand the Gauss-Jordan Elimination method and its role in solving linear systems.
  • Apply nested loops and array operations to perform row transformations on an augmented matrix.
  • Learn how to read a matrix from a file, store it in a 2D array, and process it efficiently in C++.

2. Problem Statement

A system of linear equations consists of multiple equations that need to be solved simultaneously. One efficient method for solving such systems is Gaussian Elimination, which transforms the system into an upper triangular form and then applies back-substitution to find the solution. In this lab, you will implement Gauss-Jordan Elimination in C++ by converting a given Python implementation into an equivalent C++ program. Gauss-Jordan Elimination is an advanced form of Gaussian Elimination that converts the system into Reduced Row Echelon Form (RREF), allowing for a direct solution without the need for back-substitution. The goal is to apply row operations on an augmented matrix representation of a linear system to systematically solve for the unknown variables.

(1) Matrix Representation of the System

A system of linear equations can be written in augmented matrix form:

\[ \begin{bmatrix} 2 & -3 & 1 & \vert & -22 \\ 7 & 9 & -3 & \vert & 14 \\ 6 & 7 & 2 & \vert & 91 \end{bmatrix} \]

Here, the left part of the matrix represents the coefficients of the variables, while the rightmost column represents the constants on the right-hand side of the equations.

After Applying Gauss-Jordan Elimination, the matrix is transformed into Reduced Row Echelon Form (RREF):

\[ \begin{bmatrix} 1 & 0 & 0 & \vert & -4 \\ 0 & 1 & 0 & \vert & 11 \\ 0 & 0 & 1 & \vert & 19 \end{bmatrix} \]

(2) Steps for Gauss-Jordan Elimination

Gauss-Jordan Elimination is an extension of Gaussian Elimination that transforms a matrix into the identity matrix, making it easier to extract solutions directly. This method provides a direct solution without requiring back substitution, unlike standard Gaussian Elimination.

  • Step 1: Normalize the Pivot Element
    • Identify the pivot element at position (i, i).
    • Divide the entire row by this pivot to make the leading coefficient 1.
  • Step 2: Eliminate Other Entries in the Column
    • Modify all rows except the pivot row by subtracting a multiple of the pivot row to make all other values in the column zero.
  • Step 3: Repeat for Each Column
    • Loop through each row, repeating Step 1 and Step 2, until the matrix is transformed into an identity matrix.
  • Final Output:
    • The left side of the augmented matrix becomes an identity matrix, while the right side contains the solutions to the system.

(3) Algorithm Representation in Python

The following Python code demonstrates the Gauss-Jordan Elimination process:

        n_row, n_col = aug_mat.shape

        for i in range(n_row):  
            divisor = aug_mat[i, i]  
        
            for j in range(n_col):  
                aug_mat[i, j] /= divisor  
        
            for j in range(n_row):  
                if j != i:  
                    multiplier = -aug_mat[j, i]  
                    aug_mat[j, :] += multiplier * aug_mat[i, :]
