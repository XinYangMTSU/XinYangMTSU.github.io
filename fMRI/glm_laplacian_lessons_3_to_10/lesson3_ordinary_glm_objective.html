Lesson 3 — Ordinary GLM Objective

How ordinary first-level GLM estimates beta maps by minimizing prediction error.

1. What ordinary GLM is trying to do

At every vertex, we have measured BOLD values \(y\). We also have predicted time courses in \(X\). The GLM chooses beta weights so that \(X\beta\) looks as close as possible to \(y\).

\[\hat{\beta}=\arg\min_\beta \frac12\|y-X\beta\|_2^2\]
Measured BOLD and fitted prediction

2. The error is the vertical distance

For each time point: residual = measured value − predicted value.

\[\varepsilon = y-X\beta\]
Residuals over all 128 time points
Least squares chooses beta values that make these residuals small overall.

3. From one vertex to all vertices

For all vertices together, replace \(y\) with \(Y\) and \(\beta\) with \(B\).

\[\hat B=\arg\min_B\frac12\|Y-XB\|_F^2\]

4. Why the closed-form solution looks like this

\[\hat B_{\rm OLS}=(X^\top X)^{-1}X^\top Y\]

Zero-abstraction reading:

\(X^\top Y\)
how each design column matches data
→
\(X^\top X\)
how columns overlap each other
→
\(\hat B\)
corrected beta weights
If two design columns are similar, \(X^\top X\) corrects for that overlap.

5. Key takeaway

Ordinary GLM finds the beta maps that best reconstruct the observed BOLD data, but it does not yet use cortical neighborhood information.
