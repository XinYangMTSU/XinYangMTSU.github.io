2. Events are placed on this 5-minute clock

During the scan, the subject hears, sees, reads, computes, or presses buttons. Each event happens at a particular time. These events are the raw material for the design matrix.

Illustrative event schedule across the 5-minute scan
Important: Events are not the final design matrix columns. They are only the starting dots/spikes.

3. One event does not create an instant brain signal

The fMRI BOLD response is delayed and smooth. A 1-second event becomes a bump lasting many seconds.

4. Build one design column: audio_left

Take only one condition. Mark when it happens. Then replace each event spike by an HRF-shaped bump. The result is one column of the design matrix.

Step A: audio_left raw event column, 128 points
Step B: audio_left after HRF, 128 points
This HRF-smoothed 128-number vector is one design-matrix column.

5. Why event spacing matters

If events are close together, their HRF bumps overlap. The design matrix records that overlap.

Two example condition columns over 128 points

When two columns look too similar, the GLM has difficulty deciding which condition caused the measured BOLD signal. This is why experiment timing matters.

6. Where drift columns come from

Some BOLD changes are slow scanner/background trends, not task responses. Drift columns give the GLM a way to explain these slow trends without confusing them for task activation.

A slow drift regressor over the same 128 rows

\[\text{Design matrix} = \begin{bmatrix} \text{row 1 predictions} \\ \text{row 2 predictions} \\ \vdots \\ \text{row 128 predictions} \end{bmatrix}\]
Memory hook: rows answer “when?”; columns answer “what predicted signal shape?”

8. How the GLM uses this

At one surface vertex, you also have 128 measured BOLD values. The GLM asks which mixture of design columns best reconstructs those measured values.

\[ \text{measured BOLD} \approx \beta_1(\text{audio-left column}) + \beta_2(\text{audio-left derivative}) + \cdots + \beta_{18}(\text{drift 2}) \]
Final idea: the design matrix is not the result. It is the set of candidate explanations used to estimate the result.
