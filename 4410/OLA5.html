OLA 5: Your Website Title Here

Course: CSCI 4410/5410 Web Technologies

Instructor: Dr. Xin Yang

Due date: April 12th, 2026 (11:59 PM) (Sunday)

Objective:

In this assignment, you will build a web application/website using the concepts learned in class. Your task is to design and implement a functional website with a clear structure, interactive features, and responsive styling. You should organize your content carefully, apply appropriate web technologies, and create a polished user experience. By completing this project, you will strengthen your skills in planning, implementation, and reflection.

Workflow Steps (3 Stages)

You must submit all three stages:

Stage 1: Plan the Outline of Structure
Stage 2: Website Implementation
Stage 3: Reflection: what changed and why

Stage 1: Plan the Outline of Structure (5 Points)

Before building your website, plan the structure of your project.

What to submit

  1. Option A: A clear photo of a hand-drawn outline, OR
  2. Option B: A text file named outline5.txt

Outline requirements

Your outline must show the structure of your website/application and include:

  1. A page title area (H1)
  2. A navigation bar with links
  3. Main page layouts
  4. Key sections/components
  5. Buttons/actions area
  6. A display/content area
  7. A footer or bottom section

Example outline (text)

Top
- H1: Website Title

Section 1: Navigation
- Home link
- Page 1 link
- Page 2 link

Page 1
- Heading
- Main content area

Page 2
- Heading
- Interactive/content section
- Buttons/actions

Bottom
- Footer
- Reflection (Stage 3) in outline5.txt

Stage 2: Website Implementation (90 Points)

Step 1: Project Setup

Create your project folder structure and organize your files properly. 

/your-project
│── index.html
│── /css
│   ├── style.css
│── /js
│   ├── script.js
│── /images

Step 2: Page Structure and Content (30 Points)

In this step, you will build the main structure and content of your website. 

(1) Create the required pages or sections. (15 points)
    a. Include headings and organized content.
    b. Ensure each page/section serves a clear purpose.

(2) Use semantic HTML elements properly. (10 points)
    a. Use header, nav, main, section, footer where appropriate.
    b. Keep the code clean and readable.

(3) Ensure the layout is well-structured and functional. (5 points)
    a. Organize content clearly.
    b. Use meaningful names for classes/IDs.

Step 3: Navigation and User Experience (16 Points)

This step focuses on making your website easy to navigate and pleasant to use. 

(1) Add a navigation menu with working links. (6 points)
    a. Users should be able to move between sections/pages easily.

(2) Ensure a logical flow of information. (5 points)
    a. Arrange content in a user-friendly order.

(3) Handle missing or invalid pages/links if needed. (5 points)
    a. Include a fallback or clear error message if applicable.

Step 4: Interactivity / Functionality (18 Points)

Add interactive or dynamic features required for your website. 

(1) Implement required user interactions. (6 points)
    a. Buttons, forms, menus, or other features as needed.

(2) Use JavaScript appropriately for dynamic behavior. (6 points)
    a. Update content or handle events correctly.

(3) Validate user input if forms are included. (6 points)
    a. Prevent empty or invalid submissions.
    b. Show helpful error messages where needed.

Step 5: Content Display / Actions (16 Points)

This step focuses on how content is shown and how user actions affect the page. 

(1) Display information clearly and effectively. (6 points)
    a. Organize text, images, or data in a readable way.

(2) Implement any required action buttons or controls. (6 points)
    a. Buttons should work correctly and update the interface as needed.

(3) Ensure updates appear immediately in the UI. (4 points)
    a. Changes should be reflected clearly for the user.

Step 6: Styling & UI Enhancement (10 Points)

Please create your own unique layout and styling instead of copying an example directly. 

(1) Apply CSS to improve appearance. (4 points)
    a. Create a consistent layout.
    b. Use CSS classes for styling.

(2) Style navigation, buttons, forms, and content areas. (3 points)
    a. Ensure a polished visual design.
    b. Add hover effects where appropriate.

(3) Ensure responsiveness. (3 points)
    a. The website should work well on desktop and mobile.

Stage 3: Reflection (5 Points)

At the bottom of your outline5.txt, include a section titled Reflection. Answer each question in 2–3 sentences:

  1. What changed between your Stage 1 outline and your final implementation? Why did you change it?
  2. What was the hardest part to implement correctly, and why?
  3. Describe one mistake you made and how you fixed it.

Step 7: Submission Instructions

Please compress your project folder into a single ZIP archive and submit it through D2L. If your project contains large dependency folders, you may remove them before creating the ZIP archive.

Please include:

  1. Stage 1: outline5.txt or a clear photo of your hand-drawn outline
  2. Stage 2: your website/project files
  3. Stage 3: reflection written at the bottom of outline5.txt
